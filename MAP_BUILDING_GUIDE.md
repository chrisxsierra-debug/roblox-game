# Map Building Guide

The world is generated by `WorldService` + `PlotFactory` (Track C) when no authored map is present
(§22). You can replace the placeholder generator with a hand-built map at any time.

## Using an authored map instead of the generator
1. Build your map under `Workspace/Map` (anchored).
2. Add 8 plot anchors named consistently (e.g. `Workspace/Map/PlotAnchors/Plot1..Plot8`), each a Part
   whose `CFrame` is where that plot's origin should sit, spaced for large final creatures.
3. `WorldService` should detect `Workspace/Map` and **skip** placeholder generation (idempotent — never
   generate duplicates).
4. Keep generated content inside a clearly named folder (e.g. `Workspace/GeneratedWorld`) so it's easy
   to clear.

## Each plot must contain (§11)
- A `PlotId` + `OwnerUserId` attribute (see `Constants.Attributes`) and the `Plot` tag (`Constants.Tags`).
- Owner sign, player spawn point, farm ground.
- 12 grow-slot positions tagged `GrowSlot` with a `SlotIndex` attribute (first 2 active, rest locked
  markers).
- Rebirth altar/button, shop sign, decorative border, and a placeholder for the future raid entry.

## Performance (§29)
Anchored Parts only; cache plot references; avoid repeated `GetDescendants()` in hot paths; keep
instance counts reasonable; space plots so giant-mutation creatures don't overlap neighbours.
